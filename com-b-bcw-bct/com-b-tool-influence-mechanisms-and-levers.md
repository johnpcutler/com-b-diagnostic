# OCR: How Tools Influence COM-B

## Description
This table explains how tools can influence each part of the COM-B model. For every COM-B element it shows what tools can realistically affect, the mechanisms through which tools produce that influence, and practical examples of tool levers. It provides a simple way to understand how technology can shape capability, opportunity, and motivation across knowledge, skills, workflows, norms, identity, habits, and more.

## Extracted Content

| COM-B Element | What Tools Can Influence | How Tools Influence It (Mechanisms) | Examples of Tool Levers |
|---|---|---|---|
| Psychological Capability (PC) knowledge, mental models, cognitive clarity | What people understand, how they interpret tasks, share definitions | Clarification (canonical templates, glossaries); Instruction (inline guidance, examples); Sensemaking Support (visualizations, maps, comparisons); Cognitive Offloading (structured forms, calculations) | In-app definitions and help; Guided workflows; Required fields that enforce shared meaning; Visual models of processes / dependencies; Checklists embedded in the UI |
| Physical Capability (PHC) skills, procedural ability, step-by-step execution | Ability to perform the task correctly & consistently | Simulation / Practice (wizards, sandboxing); Scaffolding (stepwise flows); Skill Transfer (pattern libraries, templates); Error Reduction (constraint-based inputs) | Structured multi-step workflows; Auto-validation; Training modes / preview modes; Pattern libraries embedded in the tool |
| Physical Opportunity (PO) time, resources, workflow enablers | Removes environmental barriers to acting | Automation (removes manual load); Integration (eliminates context switching); Streamlining (one-click routines); Reduction of Resource Cost (lower effort, lower time) | Integrations with source systems; Auto-sync data; One-click report generation; Pre-filled forms/tasks |
| Social Opportunity (SO) norms, permission, legitimacy, social cues | Influences whether people feel allowed, supported, expected to behave this way | Shared Visibility (transparency reduces doubt); Social Proof (see others doing it); Role Alignment (permissions, workflows); Depoliticization (neutral shared views) | Shared dashboards; Activity feeds; "What good looks like" examples; Cross-team visibility of work; Neutral, shared workspaces |
| Reflective Motivation (RM) deliberate reasoning, values, cost/benefit, identity | Shapes whether people choose to do the behavior intentionally | Persuasion (making value visible); Feedback Loops (connecting action to outcomes); Identity Reinforcement (naming, framing, narrative); Norm Encoding (rituals, cadences) | Outcome dashboards; Before/after comparisons; Workflows that encode best practices; Ritual prompts (weekly review, planning cycles) |
| Automatic Motivation (AM) habits, impulses, defaults, emotional ease | Creates habitual or effortless engagement | Defaults (pre-selected choices); Cues & Reminders (timely prompts); Positive Reinforcement (streaks, progress markers); Friction Reduction (effortless repetition) | Default templates; Notifications at the right moment; Progress meters, streaks; Auto-captured actions that build consistency |
