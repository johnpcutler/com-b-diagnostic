# Team Context Template

> This is a template. Copy this file to `user-context/` at the root of your workspace (for example, `user-context/team-context.md`), then fill it out.
> You can rename the copied file for your environment (for example, `team-context-platform.md`).
> If a section is unknown, leave it blank rather than guessing. The agent should treat this as background assumptions to refine, not fixed truth.

## Local team norms

- How are commitments made and renegotiated?
- How does the team handle conflict, quality concerns, and missed deadlines?
- Which implicit rules shape behavior more than formal process?

## Tech stack and tooling constraints

- Which systems, platforms, or tools are non-negotiable?
- Where are there legacy constraints, access issues, or reliability problems?
- What tooling gaps repeatedly block or slow expected behavior?

## Typical rituals and workflows

- What recurring rituals structure the work (planning, standup, review, retro)?
- Which checkpoints are required before work can move forward?
- Where do workflows depend on manual coordination or informal approvals?

## Current workload and slack capacity

- Is the team at sustainable load or chronic overload?
- What share of time goes to planned work vs interruption/reactive work?
- How much true slack exists for learning, process improvement, or experiments?

## Local leadership style

- How directive vs delegated is local leadership?
- How are priorities clarified, changed, and enforced?
- How does leadership respond to bad news, uncertainty, or dissent?
