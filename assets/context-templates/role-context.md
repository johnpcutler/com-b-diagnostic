# Role Context Template

> This is a template. Copy this file to `user-context/` at the root of your workspace (for example, `user-context/role-context.md`), then fill it out.
> You can rename the copied file for your environment (for example, `role-context-staff-engineer.md`).
> If a section is unknown, leave it blank rather than guessing. The agent should treat this as background assumptions to refine, not fixed truth.

## Core responsibilities

- What is this role officially accountable for?
- What work does this role absorb in practice that is not in the formal remit?
- Which responsibilities are ambiguous or frequently contested?

## How performance is actually measured

- What formal metrics, goals, or scorecards matter most?
- What informal signals drive evaluation (visibility, responsiveness, fire-fighting)?
- Which metrics create pressure that conflicts with desired behavior?

## Typical cross-functional dependencies

- Which teams or roles must coordinate for this role to succeed?
- Where do handoffs stall (inputs, approvals, ownership boundaries)?
- What dependencies are most politically sensitive or unreliable?

## Common pressure points

- What recurring time sinks or interruptions affect this role?
- Where does accountability exceed authority?
- Which situations create role overload, burnout risk, or defensive behavior?
