# Behavior JTBD Maturity: Diagnostic Cycle

## What this framework is
This spectrum describes the current state of a critical organizational behavior: how reliably it happens, how aligned people are in performing it, and what forces help or hinder it. It ranges from behaviors that are fully ritualized and stable to those that are contested or not even conceptually defined.

As you move across the spectrum, the behavior can become less practiced, less consistent, and more vulnerable to friction, misalignment, or political forces.

These panels map the primary and secondary blockers that prevent a behavior from moving forward at each stage of the spectrum. They use the COM-B model to diagnose why the behavior is stuck (Capability, Opportunity, Motivation), and whether the roots are cognitive, environmental, social, or habitual.

Each behavior state has:
- **Primary blockers:** the dominant forces preventing progress; highest-leverage constraints.
- **Secondary blockers:** contributing forces that reinforce the problem and add friction.

Together, these panels provide a diagnostic foundation for designing interventions, choosing tool levers, and shaping change strategies that target the specific COM-B elements holding behavior in place.

---

## The 7-state behavior cycle

### Suggested cycle flow
`Contested / Undefined` -> `Aspirational Only` -> `Weakly Realized` -> `Partially Realized / Inconsistent` -> `Realized but Friction-Filled` -> `Fully Realized & Stable` -> `Actively Suppressed` -> back to `Contested / Undefined`

This is presented as a cycle (not a one-way ladder) because behavior can advance, stall, or regress depending on shifts in incentives, norms, clarity, and operating conditions.

---

## State 1: Fully Realized & Stable
**State signal:** Ritualized, reliable, trusted.

Behavior is happening consistently, predictably, and reliably. It has become a ritual, habit, or embedded system.

Characteristics:
- Strong norms
- Clear ownership
- Predictable outputs
- Change feels risky
- Pain may exist but does not threaten the ritual

**Primary blockers (COM-B):**
- Motivation -> Reflective -> Risk aversion / loss aversion
- Opportunity -> Physical -> Environmental friction (manual, expensive workflows)
- Opportunity -> Social -> Norm pressure to preserve predictability

**Secondary blockers (COM-B):**
- Capability -> Psychological -> Procedural knowledge gap (how to automate safely)
- Motivation -> Automatic -> Habit inertia (status quo bias)
- Opportunity -> Physical -> Tooling limitations that make change risky

**High-level tool strategy:**
- Respect ritual; reduce risk of disruption (PO, RM -> ER, EN)
- Preserve the ritual while reducing invisible cost
- Maintain fidelity, ensure safety, lower manual burden, automate without altering meaning

**Tool levers:**
- Primary 1: Safe sandboxes and reversible steps (RM -> PE, EN)
- Primary 2: Automate painful steps (PO -> ER)
- Primary 3: Preserve cadence and norms (SO -> MO)
- Secondary 4: Embedded guidance builds confidence (PC -> TR, ED)
- Secondary 5: Defaults form low-effort shifts (AM -> Habit / INC)
- Secondary 6: Audit trails increase trust (RM -> PE, EN)

---

## State 2: Realized but Friction-Filled
**State signal:** Mandatory but painful; environment is the enemy.

Behavior consistently happens, but the environment makes it painful.

Characteristics:
- Legacy systems
- Workarounds
- Manual effort
- Compliance-driven
- Little joy, lots of necessity

**Primary blockers (COM-B):**
- Opportunity -> Physical -> Tool/system constraints (legacy systems)
- Opportunity -> Physical -> Limited time/resource availability
- Opportunity -> Physical -> Workflow friction (workarounds, manual steps)

**Secondary blockers (COM-B):**
- Motivation -> Automatic -> Learned helplessness ("this is just how it is")
- Capability -> Psychological -> Overload / working memory strain
- Opportunity -> Social -> Cultural acceptance of inefficiency

**High-level tool strategy:**
- Remove friction and overhead (PO -> ER, EN)
- Remove operational drag from an already accepted workflow
- Streamline, smooth, automate, unblock, speed up what people already agree on

**Tool levers:**
- Primary 1: Integrations remove bottlenecks (PO -> ER)
- Primary 2: Standard routines (PO -> ER)
- Primary 3: Standardize to reduce variance (PO -> ER)
- Secondary 4: Visibility of bottlenecks (RM -> PE)
- Secondary 5: Smoother routines become defaults (AM -> Habit)
- Secondary 6: Dashboards expose inefficiency (RM -> PE)

---

## State 3: Partially Realized / Inconsistent
**State signal:** Exists in pockets; lacks alignment.

Behavior happens, but unevenly; different teams or people do it differently.

Characteristics:
- Process variation
- No canonical method
- Conflicts of interpretation
- Uneven capability across teams
- Local optimizations, not a shared system

**Primary blockers (COM-B):**
- Capability -> Psychological -> Mental-model variation
- Capability -> Psychological -> Lack of procedural knowledge ("how to do it correctly")
- Opportunity -> Social -> Local norms / team-level divergence

**Secondary blockers (COM-B):**
- Motivation -> Reflective -> Low belief in a single "right way"
- Capability -> Psychological -> Insufficient training or shared language
- Opportunity -> Physical -> Tools do not enforce consistency

**High-level tool strategy:**
- Standardize meaning and flow (PC, SO -> ED, MO, ER)
- Standardize meaning, flow, and expectations across the organization
- Define "the way," enforce consistency, unify semantics, codify tacit knowledge

**Tool levers:**
- Primary 1: Canonical templates define "the way" (PC -> ED)
- Primary 2: Guided workflows unify skill (PC -> TR)
- Primary 3: Embed shared language (PC -> ED)
- Secondary 4: In-context nudges (AM -> ER)
- Secondary 5: Consistent flow becomes easiest path (AM -> Habit)
- Secondary 6: Comparison views reveal inconsistency (PC -> ED)

---

## State 4: Weakly Realized
**State signal:** Agreed and valued, but continually displaced.

Everyone agrees it matters, but it keeps getting displaced.

Characteristics:
- Not a knowledge problem
- Not a commitment problem
- Loses to urgency, fire drills, scarcity
- Creates guilt and a backlog of "shoulds"

**Primary blockers (COM-B):**
- Opportunity -> Physical -> Lack of time / competing urgencies
- Motivation -> Automatic -> Habit competition (other habits crowd it out)
- Opportunity -> Physical -> High activation energy (too many steps)

**Secondary blockers (COM-B):**
- Motivation -> Reflective -> Short-term focus vs long-term value
- Capability -> Psychological -> Limited self-regulation structures (no triggers)
- Opportunity -> Social -> Culture prioritizes fire drills over proactive work

**High-level tool strategy:**
- Make starting easy; reduce forgetting (PO, AM -> ER, EN)
- Reduce activation energy and make the behavior easier to start and sustain
- Provide cues, scaffolds, reminders, nudges, templates

**Tool levers:**
- Primary 1: One-click starters lower activation cost (PO -> ER)
- Primary 2: Reminders and cues (AM -> ER)
- Primary 3: Make long-term value salient (RM -> PE)
- Secondary 4: Auto-capture reduces burden (PO -> ER, EN)
- Secondary 5: Streaks and small wins (AM -> INC)
- Secondary 6: Visible progress encourages continuation (RM/AM -> PE)

---

## State 5: Aspirational Only
**State signal:** Identity-level desire without practice.

The behavior is desired or politically valuable, but not practiced.

Characteristics:
- Leadership vision statements
- Cultural aspiration
- Branding and identity motives
- Small symbolic attempts
- No actual habit infrastructure

**Primary blockers (COM-B):**
- Motivation -> Reflective -> Identity gap (aspiration > true commitment)
- Opportunity -> Physical -> No enabling structures or systems
- Capability -> Psychological -> Undefined skills or knowledge to begin

**Secondary blockers (COM-B):**
- Motivation -> Automatic -> Low emotional reward / no habit loop
- Opportunity -> Social -> Symbolic buy-in without behavioral reinforcement
- Capability -> Psychological -> Undefined "what good looks like"

**High-level tool strategy:**
- Turn aspiration into action (PC, RM -> ED, TR, EN)
- Transform aspiration into repeatable practice
- Make the abstract concrete, provide structure, enable early wins, bootstrap skill

**Tool levers:**
- Primary 1: Provide concrete structure (PC -> ED, TR)
- Primary 2: Visualize future desired state (RM -> PE, MO)
- Primary 3: Encode identity and aspiration (RM -> PE)
- Secondary 4: Scaffold early steps (PC -> TR)
- Secondary 5: Early wins reinforce identity (AM -> INC)
- Secondary 6: Make emerging identity visible (RM -> MO)

---

## State 6: Actively Suppressed
**State signal:** Systemic forces prevent the behavior.

Not only is the behavior not happening; explicit or implicit forces prevent it.

Characteristics:
- Political risk (doing it creates exposure)
- Turf protection ("do not step on toes")
- Misaligned incentives (reward systems punish the behavior)
- Cultural norms against it
- Learned helplessness

**Primary blockers (COM-B):**
- Motivation -> Reflective -> Political risk / threat to status
- Opportunity -> Social -> Norms that punish or discourage the behavior
- Opportunity -> Social -> Misaligned incentives

**Secondary blockers (COM-B):**
- Motivation -> Automatic -> Fear responses (threat, anxiety)
- Capability -> Psychological -> Low psychological safety to attempt the behavior
- Opportunity -> Physical -> Systems designed to block the behavior

**High-level tool strategy:**
- Make safe and depoliticized (SO, RM -> ER, MO)
- Make the behavior politically safe, neutral, and acceptable
- Create safe channels, depersonalize conflict, legitimize practice without provoking friction

**Tool levers:**
- Primary 1: Make behavior legible (SO, RM -> ER)
- Primary 2: Neutral shared views (SO -> MO)
- Primary 3: Private safe workspaces (RM -> EN)
- Secondary 4: Clear roles and permissions (SO -> ER)
- Secondary 5: Transparent workflow reduces avoidance (AM -> ER)
- Secondary 6: Visibility aligns incentives (SO -> INC)

---

## State 7: Contested / Undefined
**State signal:** No shared definition; conceptual ambiguity.

People cannot agree on, or clearly describe, what the behavior actually is. This is not disagreement about how to do it; it is disagreement about what it even means.

Characteristics:
- Fundamental definitional ambiguity
- Competing framings (for example: "What is discovery?", "What counts as a customer?", "What does strategy mean?")
- No shared language
- Same words used with different meanings
- Meetings derail into debates about definitions
- Extremely high cognitive load
- Alignment is impossible because the conceptual object is unstable

**Primary blockers (COM-B):**
- Capability -> Psychological -> Mental-model divergence (core definitional conflict)
- Capability -> Psychological -> Lack of conceptual knowledge / shared language
- Motivation -> Reflective -> Identity-driven attachment to competing definitions

**Secondary blockers (COM-B):**
- Opportunity -> Social -> Semantic drift (same words, different meanings)
- Opportunity -> Social -> Intergroup friction / turf over ownership of definitions
- Capability -> Psychological -> Cognitive overload / ambiguity tolerance

**High-level tool strategy:**
- Create shared clarity (PC, SO -> ED, ER)
- Create shared clarity and converge on coherent behavior definition
- Surface differences, reveal mental models, align terminology, reduce semantic drift

**Tool levers:**
- Primary 1: Shared definitions (PC -> ED)
- Primary 2: Visualize divergent models (PC -> ED)
- Primary 3: Required fields force clarity (PC -> ED)
- Secondary 4: Conceptual scaffolds (PC -> ED)
- Secondary 5: Default templates crystallize patterns (AM -> Habit)
- Secondary 6: Visualized disagreement accelerates convergence (PC, SO -> ED, MO)

## COM-B + BCW abbreviations referenced

### COM-B codes
- PC = Psychological Capability
- PHC = Physical Capability
- PO = Physical Opportunity
- SO = Social Opportunity
- RM = Reflective Motivation
- AM = Automatic Motivation

### BCW intervention functions
- ED = Education
- TR = Training
- PE = Persuasion
- INC = Incentivization
- COE = Coercion
- RE = Restriction
- ER = Environmental Restructuring
- MO = Modeling
- EN = Enablement

---

## Practical use
Use this cycle to diagnose where a behavior currently sits, identify the COM-B blockers at that state, and choose the highest-leverage tool interventions and change approach for behavioral readiness.
